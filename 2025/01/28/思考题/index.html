思考题

着重学习思考如何使用axios进行前后端数据传输

​ 1. GET 请求：从服务器获取数据，例如加载列表、获取用户信息。

​ 2. POST 请求：向服务器发送数据，例如提交表单、用户登录。

​ 3. 数据格式：

​ • JSON 是最常用的格式，适合结构化数据。

​ • 表单格式用于传统提交方式。

​ • 文件上传需使用特殊格式（如 FormData）。

​ 4. 全局配置：可以设置基础路径、请求头和超时时间，减少重复代码。

​ 5. 常见场景：登录认证、动态获取内容、表单提交、文件上传等。

​ 6. 错误处理：需要针对网络错误、后端响应错误进行统一处理。

vue相比原生三件套有什么特点，体现在哪里？

​ 1. 组件化：代码模块化，结构清晰，方便复用。

​ 2. 响应式：数据和视图自动同步，无需手动操作 DOM。

​ 3. 指令简化：使用如 v-for、v-if 等指令，让操作 DOM 更直观。

​ 4. 双向绑定：用 v-model 实现数据和表单的实时同步。

​ 5. 性能优化：通过虚拟 DOM 提升渲染效率。

​ 6. 生态系统：内置路由和状态管理工具，开发更高效。

学习了vue之后，之前写的小米官网有哪些可以优化的地方

​ 1. 组件化开发

​ • 问题：页面代码重复，结构不清晰，逻辑混杂。

​ • 优化：将页面拆分为独立的组件（如导航栏、商品列表、轮播图等），实现模块化开发，提高代码复用性。

​ 2. 数据驱动视图

​ • 问题：手动操作 DOM 更新数据，易出错且效率低。

​ • 优化：使用 Vue 的响应式数据，自动更新视图，减少 DOM 操作。

​ 3. 动态路由

​ • 问题：页面跳转通过静态链接或复杂的 JS 实现。

​ • 优化：使用 Vue Router 管理页面跳转和嵌套路由，实现单页应用体验。

​ 4. 状态管理

​ • 问题：数据共享通过全局变量，易导致逻辑混乱。

​ • 优化：使用 Pinia 管理全局状态，统一管理购物车、用户信息等数据。

​ 5. 双向数据绑定

​ • 问题：表单数据需要手动绑定和同步，代码复杂。

​ • 优化：使用 v-model 实现表单数据与视图的双向绑定。

​ 6. 指令优化

​ • 问题：通过 JS 手动控制元素的显示隐藏和样式切换。

​ • 优化：使用 Vue 的指令（如 v-if、v-show、v-bind）简化逻辑。

​ 7. 性能优化

​ • 问题：全量 DOM 操作或刷新页面，影响性能。

​ • 优化：通过 Vue 的虚拟 DOM 和组件更新机制，仅更新需要变化的部分。
