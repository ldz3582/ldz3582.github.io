重点了解使用vue比原生三件套有哪些好处、ref和reactive的响应式设计的用法与作用（重中之重）、watch监视的基本使用与应用场景（1-21）

使用 Vue 比原生三件套的好处

​ • 模块化开发：Vue 的组件化设计使得代码结构清晰，可复用性高。

​ • 双向绑定：通过 v-model 实现数据和视图的同步，减少 DOM 操作。

​ • 响应式系统：自动更新视图，无需手动操作 DOM。

​ • 指令简化操作：如 v-for、v-if 提供直观的逻辑控制。

​ • 生态系统：集成了 Vue Router、Pinia 等工具，简化开发流程。

​ • 性能优化：虚拟 DOM 提高了渲染效率。

Ref–基本类型或者对象

reactive:主要是对象

Watch:监听一个值watch(refVar, (newVal, oldVal) => { … })

监听多个值watch([var1, var2], ([new1, new2], [old1, old2]) => { … })。

重点了解prop属性的使用、vue的生命周期和生命周期钩子函数（重点）、vue路由的使用与理解（重点）、pinia的作用和使用方式（22-51

回答：

props是传递数据的，一般就是父子之间吧，子组件一般是用defineProps定义接收props

vue生命周期分为

创建阶段: setuo,onBeforeMount

挂载阶段：onMounted(DOM)

更新阶段：onBeforeDate,onUpdated(响应式数据更新)

销毁阶段：onBeforeUnmount,onUnmounted（组件销毁前后）

钩子函数

onMounted：初始数据加载、DOM 操作。

onUnmounted：清理定时器、移除监听器。

onUpdated：数据变化后触发。

路由

就是路径参数与API

Router,push()->编程式导航

useRoute 获取当前路由信息

useRouter:执行导航操作

pinia

State:定义全局状态

Getters:计算属性，派生状态

Actions:修改状态

使用方式在vue(ts)里面有步骤呈现

重点学习父子组件和兄弟组件的通信方式，插槽的作用与使用方式

Answer：

父子组件通信：父-》子 props,子-〉父，$emit

兄弟之间通信：vue2:Vuex,vue3:pinia

插槽：实现父组件向子组件传递HTML结构，提升组件灵活性

基础插槽，具名插槽，作用域插槽
