第一步：需求分析与设计

​ • 目标：明确每个功能模块的需求和页面设计。

​ • 内容：

​ • 功能分析：登录页面、AI大模型交互页面、用户中心。每个模块需要做什么，功能如何实现。

​ • 设计规划：明确页面布局，决定每个功能的位置和交互方式，思考用户使用的流畅性。

第二步：页面设计与布局

​ • 目标：在设计工具（如 Figma 或 Sketch）中创建页面的原型和布局。

​ • 内容：

​ • 登录页面设计：基本的表单，输入框（账号、密码）、按钮（登录）、错误提示。

​ • AI大模型交互页面设计：对话框区域、输入框、发送按钮、流式显示区域（支持图片、文本等）。

​ • 用户中心设计：修改密码、修改个人资料、登出按钮等。

第三步：搭建基本的前端框架

​ • 目标：初始化项目并搭建项目基础框架，使用你熟悉的技术栈（例如：HTML、CSS、JavaScript 或 Vue/React）。

​ • 内容：

​ • 创建页面的静态框架，使用 HTML/CSS 搭建页面布局（不涉及功能时，只实现静态页面）。

​ • 使用 Git 进行版本控制，并创建一个 GitHub 仓库，确保每次修改都能提交版本。

第四步：实现登录功能

​ • 目标：实现基本的登录页面功能，模拟简单的账号密码匹配。

​ • 内容：

​ • 使用 JavaScript 实现用户输入的验证（判断账号密码是否匹配）。

​ • 使用本地存储（如 localStorage 或 sessionStorage）模拟登录状态。

第五步：接入 Coze API，实现对话功能

​ • 目标：使用 Coze API 实现用户输入与 AI 大模型的交互。

​ • 内容：

​ • 学习如何调用 Coze API，了解它如何接收请求、返回结果。

​ • 实现流式对话功能：用户输入后，Coze API 返回内容并逐行展示。

​ • 支持文本和图片等不同格式的显示。

​ • 实现新增对话窗口的功能：每个对话保持独立且能保留历史对话，且可以编辑、删除。

第六步：用户中心功能实现

​ • 目标：实现用户资料修改和登出功能。

​ • 内容：

​ • 修改用户名、密码功能。

​ • 实现登出，清除登录状态。

第七步：美化与细节优化

​ • 目标：让页面更加美观与易用，提升用户体验。

​ • 内容：

​ • 调整布局，优化颜色、字体、按钮样式。

​ • 使页面具有响应式设计，确保在移动端和桌面端都能良好展示。

第八步：代码规范与工具使用

​ • 目标：整理代码，确保符合编码规范。

​ • 内容：

​ • 使用 ESLint 检查代码规范。

​ • 确保文件结构清晰，代码注释完整。

第九步：测试与功能完善

​ • 目标：检查所有功能是否按预期工作，并进行测试。

​ • 内容：

​ • 手动测试：测试登录功能、对话框输入输出、用户修改和登出等功能。

​ • 修复 bug，优化交互体验。

第十步：提交与总结

​ • 目标：将最终代码提交，准备设计稿件和提交材料。

​ • 内容：

​ • 完成页面设计稿（Figma 或其他工具）。

​ • 整理项目文件，提交 GitHub 仓库，并编写简要的项目说明。
