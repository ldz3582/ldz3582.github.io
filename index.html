##介绍

Promise 是 JavaScript 中的一种异步编程解决方案，它代表了一个异步操作的最终完成（或失败）及其结果值的表示。简单来说，Promise 用来处理异步操作的结果，它的核心目的是解决回调地狱（callback hell）问题，并让代码更加简洁、易于维护。

关键词：异步操作哈

状态

Promise 的状态

一个 Promise 实例有三种状态：

​ 1. Pending（待定）：初始状态，既不是成功，也不是失败。

​ 2. Fulfilled（已完成）：异步操作成功完成，且返回了结果。

​ 3. Rejected（已拒绝）：异步操作失败，且返回了错误。

这三种状态之间的转换是单向的：

​ • 从 Pending 状态可以转为 Fulfilled 或 Rejected。

​ • 一旦状态发生改变（从 Pending 到 Fulfilled 或 Rejected），它就不可再变了。

语法

创建 Promise

要创建一个 Promise 对象，可以使用 new Promise() 构造函数，它接受一个回调函数，这个回调函数有两个参数：resolve 和 reject。

​ • resolve(value)：表示操作成功，传递成功的结果。

​ • reject(reason)：表示操作失败，传递失败的原因。

Promise 支持链式调用，可以在一个 Promise 的 then 方法后接着另一个 then 或 catch。这样可以避免嵌套回调函数（即回调地狱）。

Promise 的常用方法

​ • then(onFulfilled, onRejected)：Promise 成功或失败后的处理函数。

​ • onFulfilled(value)：Promise 被 resolve 后执行的回调。

​ • onRejected(reason)：Promise 被 reject 后执行的回调。

​ • then() 方法会返回一个新的 Promise，可以继续链式调用。

​ • catch(onRejected)：处理 Promise 的拒绝（失败）情况，实际上是 .then(undefined, onRejected) 的简写。

​ • finally(onFinally)：无论 Promise 成功还是失败，都会执行的回调，适合用于清理操作。

认识AJAX

AJAX（Asynchronous JavaScript and XML）是一种在不重新加载整个页面的情况下与服务器交换数据并更新部分网页内容的技术。它使得网页应用程序能够在后台异步地与服务器进行通信，提升了用户体验，使页面响应更加快速、流畅。

AJAX的核心组成部分通常包括以下技术：

​ 1. XMLHttpRequest：这是一个浏览器内建的JavaScript对象，用来异步地发送请求并接收响应。它可以发送不同类型的请求（如GET、POST等），并处理服务器的响应。

​ 2. JavaScript：用于编写逻辑来发送请求、接收响应并动态更新页面内容。

​ 3. HTML/CSS：动态更新页面内容时使用，用来显示服务器响应的数据。

​ 4. 服务器端语言（如PHP、Node.js、Java、Python等）：用于处理AJAX请求并返回相应数据。

基本的AJAX流程

​ 1. 客户端通过JavaScript创建一个XMLHttpRequest对象。

​ 2. 使用该对象向服务器发送请求。

​ 3. 服务器处理请求并返回数据（通常是JSON或XML格式）。

​ 4. JavaScript处理服务器返回的数据，并使用DOM（Document Object Model）来更新页面内容。

XML

XML（eXtensible Markup Language，扩展标记语言）是一种用于描述数据结构和传输数据的标记语言。它与HTML类似，但XML的主要目的是存储和传输数据，而不是呈现数据。

AJAX特点

AJAX优点

  • 无需刷新页面与服务器进行通信

  • 允许你根据用户事件来更新部分页面内容

AJAX缺点

  • 没有浏览历史，不能回退

  • 存在跨域问题

  • SEO不友好（网站或网页的结构、内容、代码等不利于搜索引擎优化（SEO），使得搜索引擎难以正确地索引和排名这些页面）

URL（Uniform Resource Locator）本质上就是一个资源的地址，它指示了如何在网络中找到某个资源（比如网页、图片、JavaScript 文件等）。在你的例子中，URL 是用来指定 Axios 库所在的网络位置。

URL 的结构

URL 通常由以下几部分组成：

​ 1. 协议（Protocol）：

​ • 例如：http:// 或 https://，指定了如何与服务器进行通信。

​ 2. 域名（Domain）：

​ • 例如：cdn.bootcdn.net，它是一个服务器的地址，告诉浏览器要连接哪个服务器。

​ 3. 路径（Path）：

​ • 例如：/ajax/libs/axios/0.21.1/axios.min.js，它指向服务器上具体资源的位置。

​ 4. 查询参数（Query parameters）（可选）：

​ • 例如：?v=1，可以附加一些参数来控制资源的获取方式。

所以，像你在代码中使用的 https://cdn.bootcdn.net/ajax/libs/axios/0.21.1/axios.min.js 就是 Axios 库的一个 URL，它包含了协议、域名和路径，指示了从 CDN 服务器加载 Axios 的位置。

为什么需要 HTTP 请求？

在现代 Web 开发中，客户端（通常是浏览器）和服务器之间的通信是不可或缺的。前端应用通过 HTTP 请求与后端 API 交互，获取数据、发送数据，或者执行某些操作。HTTP 请求就是这种通信的桥梁

GET 请求（获取数据）

​ • 用途：GET 请求用于从服务器获取资源或数据。它是最常用的请求类型。

​ • 例子：你在浏览器中访问一个网页时，浏览器实际上发出的是一个 GET 请求来请求该页面的数据。

1.2 POST 请求（提交数据）

​ • 用途：POST 请求用于向服务器提交数据，通常是向服务器创建新的资源或提交表单数据。

​ • 例子：你在注册页面输入信息并提交表单时，浏览器会发出一个 POST 请求，将表单数据发送到服务器进行处理（如创建新用户）。

1.3 PUT 请求（更新数据）

​ • 用途：PUT 请求用于更新服务器上的资源。它通常用于替换资源的整个内容。

​ • 例子：假设你想更新用户的资料信息，可以通过 PUT 请求将新的数据发送给服务器。

1.4 DELETE 请求（删除数据）

​ • 用途：DELETE 请求用于删除服务器上的资源。它通常用于删除某个特定的资源。

​ • 例子：你可能在一个用户管理界面中提供删除用户的功能，当用户点击删除时，发出一个 DELETE 请求来从服务器删除该用户。

拦截器是什么东西？？什么意思

截器（Interceptor）是一个在发送请求或接收响应时可以进行处理或修改的函数。它们提供了一种在请求被发送到服务器或响应到达前对其进行拦截并进行自定义处理的机制。

在 Axios 中，拦截器允许我们在请求发送之前对请求进行修改，或者在响应到达时对响应进行处理。这使得我们可以在应用中集中处理认证、错误管理、日志记录、修改请求数据或响应数据等任务，而无需在每个请求中都进行这些操作。

1. 请求拦截器（Request Interceptor）

请求拦截器会在请求被发送到服务器之前执行，可以对请求进行修改，比如添加请求头、请求参数等。

2. 响应拦截器（Response Interceptor）

响应拦截器会在服务器响应数据返回给客户端之前执行。你可以在响应到达前对响应数据进行处理，比如统一处理错误、格式化数据等。

1. 客户端向服务器发送请求

在 Web 应用中，客户端通常指的是运行在浏览器中的应用，而服务器是存储数据和处理请求的后端系统。当用户在客户端（浏览器）发起某个操作时（例如点击按钮、提交表单等），客户端会向服务器发送一个 HTTP 请求。

这个请求通常会包含：

​ • 请求方法（如 GET、POST、PUT、DELETE 等）。

​ • 请求 URL（如 https://api.example.com/users）。

​ • 请求头（headers），如认证信息或内容类型等。

​ • 请求体（data 或 payload），如用户输入的表单数据、JSON 格式的数据等。

2. 服务器处理请求

当服务器接收到客户端的请求时，服务器会根据请求内容执行相应的操作，通常包括：

​ • 解析请求数据：服务器会解析请求头、请求体等内容，判断请求是否有效。

​ • 执行操作：根据请求的类型（如查询、更新、删除数据等），服务器会与数据库进行交互、执行计算等。

​ • 生成响应数据：服务器将处理结果（如查询到的用户信息、操作成功的消息等）生成响应数据。

服务器的响应数据通常是一个 JSON 格式 的对象，或者是 HTML、XML 等格式。

3. 服务器向客户端返回响应

服务器将处理结果作为响应返回给客户端。这通常是一个 HTTP 响应，它包含：

​ • 状态码（如 200 OK 表示请求成功，404 Not Found 表示资源未找到，500 Internal Server Error 表示服务器错误等）。

​ • 响应头（如返回的数据类型、缓存控制等）。

​ • 响应体（data），即服务器返回的数据，通常是 JSON、HTML 或其他格式。

离散化
首先大家最关注的问题就是什么时候用离散化 呢？

为什么要用离散化？ 离散化就是将连续或者大范围的数据映射到一个极小，连续的整数范围内的过程，在许多算法问题中，尤其是涉及到大量范围查询或者频繁更新操作时，离散化可以显著提高效率和降低时空复杂度，离散化的使用场景：

1）处理大范围的数据

2）优化空间利用

解决方案：通过离散化，将所有出现过的x,l,r进行排序并去重，映射到一个连续的较小整数范围内，优势：空间节省，提高缓存命中率。

3）简化查询与更新。

一般离散化就是要将稀疏数据映射到紧凑范围，那就会有很多人不理解排序了为什么要去重啊

这里就是解释：这样做就是为了确保每个唯一数值有唯一的索引，因为离散化本质上就是映射。

每个数值对应一个唯一的索引，索引是连续且有序的。

这样做，避免重复映射导致的错误，如果不进行去重，alls 可能出现很多相同的值，

那么有的人还会有一个问题：为什么去重不会影响求和，去重只是为了建立机制，不影响数组本身。

所以离散化本质就是映射，理解了映射这个题目就理解了。

但是还是很难理解

这是我一开始的疑问：为什么要离散化两个端点（l,r) 离散化的目的是将原始坐标映射到一个连续，较小的范围，这样做是节省空间。将所有相关坐标（插入的坐标与查询端点）放在一个数组alls中。

add使用来插入的，query是用来查询操作的

二 第二章：迭代器

迭代器是 JavaScript 中一种用于按需访问集合元素的机制。它提供了一种统一的方式来遍历集合对象（如数组、字符串、Set、Map 等）。通过迭代器，开发者可以按顺序获取集合中的元素，而无需关心底层数据结构的实现迭代器是一个对象，它符合迭代协议（Iterator Protocol）。

​ • 它提供一个 next() 方法，每次调用会返回一个包含 value 和 done 两个属性的对象：

​ • value：当前迭代的值。

​ • done：布尔值，表示是否迭代完成。

​ • false 表示还有更多元素。

​ • true 表示迭代结束。

工作原理

  • 创建一个指针对象，指向当前数据结构的起始位置

  • 第一次调用对象的next方法，指针自动指向数据结构的第一个成员

  • 接下来不断调用next方法，指针一直想后移动，直到指向最后一个成员

  • 每调用next方法返回一个包含value和done属性的对象

二 第二章：生成器

**生成器（Generator）**是 ES6 引入的一种特殊函数，用于控制函数的执行流程。生成器函数可以暂停执行，并在需要时恢复执行。生成器返回一个 迭代器对象，通过 yield 关键字生成值。

​ 1. 暂停与恢复

​ • 每次执行 gen.next()，生成器从上一次暂停的地方恢复执行，直到遇到下一个 yield 或函数结束。

​ • yield 会将一个值返回给调用者，并暂停生成器的执行。

​ 2. next() 的返回值

​ • value：当前 yield 的值（或 return 的值）。

​ • done：布尔值，表示生成器是否完成。

​ 3. 执行流程

​ • 第一次调用 next()，生成器开始执行，直到第一个 yield。

​ • 再次调用 next()，从暂停点继续执行，直到下一个 yield 或 return。

promise介绍与基本语法

Promise 是 ES6 引入的一种用于处理异步操作的对象。它提供了更加优雅的方式来组织异步代码，避免回调地狱（Callback Hell）的问题

1.1 什么是 Promise？

​ • Promise 是一个代表未来某个事件（通常是一个异步操作）最终结果的对象。

​ • 一个 Promise 有三种可能的状态：

​ 1. Pending**（进行中）**：初始状态，既没有被解决，也没有被拒绝。

​ 2. Fulfilled**（已完成）**：操作成功完成，Promise 被解决。

​ 3. Rejected**（已拒绝）**：操作失败，Promise 被拒绝。

状态一旦从 Pending 变为 Fulfilled 或 Rejected，就不可再改变

  • 基本语法

  • 实例化 Promise 对象

    Promise 是一个构造函数，接受一个执行器函数（executor）作为参数。执行器函数有两个参数：

    ​ • resolve：将 Promise 状态从 Pending 变为 Fulfilled。

    ​ • reject：将 Promise 状态从 Pending 变为 Rejected。

    语义化
    HTML语义化 是指在编写HTML代码时，通过使用具有明确含义的标签来结构化网页内容，使代码具有更好的可读性和可维护性，同时帮助浏览器、开发者、搜索引擎更准确地理解网页内容的结构和含义。
    

    为什么需要HTML语义化？
    

    ​	1.	提升代码可读性：
    

    ​	•	让开发者通过标签的名称就能直观地理解内容的用途。
    

    ​	•	方便团队协作和后续维护。
    

    ​	2.	增强SEO（搜索引擎优化）：
    

    ​	•	语义化标签帮助搜索引擎更准确地抓取和理解网页内容，提高网页排名。
    

    ​	3.	改进无障碍支持：
    

    ​	•	辅助技术（如屏幕阅读器）可以更好地解析网页内容，提升对残障用户的友好程度。
    

    ​	4.	更好的样式和行为分离：
    

    ​	•	语义化标签可以更容易被CSS和JavaScript针对性地应用样式或功能。
